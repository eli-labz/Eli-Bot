# Contributing to Eli-Bot

Thanks for your interest in contributing to Eli-Bot! This project welcomes issues, discussions, and pull requests from the community.

## Ways to Contribute

You can report bugs by opening an issue describing the problem, your environment (Windows version, Python version), and steps to reproduce. You can suggest features by opening an issue describing the use case and how it fits the project's goals. You can improve documentation by fixing typos, clarifying setup steps, or adding examples to the README or docs. You can also submit code to fix bugs, add tests, or implement new automation task packs.

## Getting Started

Fork the repository and create your branch from `main`. Install dependencies with `pip install -r requirements.txt`. Make your changes, keeping commits focused and descriptive. Run the test suite with `python -m unittest discover -s tests -p "test_*.py"`. Then open a pull request describing what you changed and why.

## Pull Request Guidelines

Keep pull requests focused on a single change where possible, and include a clear description of the motivation and approach. Add or update tests when changing behavior, and please be responsive to review feedback.

## Code of Conduct

Be respectful and constructive in issues, discussions, and reviews. We want this to be a welcoming project for contributors of all experience levels.

## Questions

If you're unsure where to start, open an issue and ask, we're happy to point you in the right direction.
